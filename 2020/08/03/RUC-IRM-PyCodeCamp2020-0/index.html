안녕하세요, 블로그 주인인 괜찮은 ‘괜찮은’입니다. 모쪼록 잘 부탁드립니다! 😉
大家好，我是博主괜찮은’GwenChanEn’。请多多关照！


우선 다른 우수한 플랫폼이 여럿 있음에도 불구하고
github 블로그를 개설한 데에는 3가지 이유가 있습니다.
虽有好多个优秀的博客平台，但我选择GitHub博客的原因如下：

  1. 개발자만의 geek한 블로그를 갖고 싶었다. 🧑‍💻
    1. 打算向后成为一名程序员，想拥有一个比较‘geek’的技术博客🧑‍💻
  2. 학습내용을 종이로 정리하는 것은 효율성 측면에서 한계가 있다. 🤯
    2. 授课过程中发现：在纸上做笔记效率有限。🤯
  3. 코로나로 계속 국내에만 있어 중국어, 영어로 작문을 할 일이 없어서 필력이 많이 떨어졌다. ✍
    3. 由于疫情特殊期间一直在国内（🇰🇷），没机会以汉语和英语写文章，感觉文笔日益后退。✍



제가 이 블로그를 개설하기 위해 참고한 블로그는 이하와 같습니다.:

  1. 超详细Hexo+Github博客搭建小白教程
  2. Markdown基本语法
  3. Connecting to GitHub with SSH
  4. Full Emoji List, v13.0
  5. Github Page와 Hexo를 통해 30분만에 기술 블로그 만들기
  6. 좋은 기술 블로그를 만들어 나가기 위한 8가지 제언
  7. MarkDown 사용법 총정리
초보자가 봐도 바로 따라할 수 있게끔 각 단계를 매우 상세하게 적어주신 godweiyang, 高鸿祥, GitHub 개발자 분들, Minhyeok Jung, Daegwon Nacyot Kim, HEROPY TECH님 께 진심으로 감사드립니다. 🙏
衷心感谢上述博客作者以及GitHub写这么详细又通俗易懂的博客，
让初始者立即随之进行操作。
